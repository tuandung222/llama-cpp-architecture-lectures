Bài 2: Đại số Tensor Layout & Memory Alignment

Bài này phân tích toán học của tensor memory layout trong GGML, từ stride computation đến SIMD alignment requirements.

1. Stride Computation trong GGML​

Với ggml_tensor có dimensions ne[4] và type TTT:

nb[0]=type_size(T)nb[0] = \text{type\_size}(T)nb[0]=type_size(T) nb[1]=nb[0]×ne[0]block_size(T)+paddingnb[1] = nb[0] \times \frac{ne[0]}{\text{block\_size}(T)} + \text{padding}nb[1]=nb[0]×block_size(T)ne[0]​+padding nb[i]=nb[i−1]×ne[i−1]for i≥2nb[i] = nb[i-1] \times ne[i-1] \quad \text{for } i \geq 2nb[i]=nb[i−1]×ne[i−1]for i≥2

Lưu ý: Với các kiểu quantization chuẩn (Q4_0, Q8_0, F32...), padding = 0 vì type_size đã bao gồm đúng kích thước block. Padding chỉ khác 0 trong một số trường hợp đặc biệt cần alignment bổ sung.

Ví dụ: Tensor F32 có shape [4096, 4096]:

  • nb[0]=4nb[0] = 4nb[0]=4 bytes (sizeof(float))
  • nb[1]=4×4096=16384nb[1] = 4 \times 4096 = 16384nb[1]=4×4096=16384 bytes
  • nb[2]=16384×4096=67,108,864nb[2] = 16384 \times 4096 = 67,108,864nb[2]=16384×4096=67,108,864 bytes

2. SIMD Alignment: Tại sao 32 bytes?​

AVX2 sử dụng thanh ghi 256-bit = 32 bytes. Aligned load (_mm256_load_si256) yêu cầu:

address≡0(mod32)\text{address} \equiv 0 \pmod{32}address≡0(mod32)

Với block quantization Q4_0 (18 bytes/block), để đảm bảo alignment:

tensor_data_offset≡0(mod32)\text{tensor\_data\_offset} \equiv 0 \pmod{32}tensor_data_offset≡0(mod32)

Đây là lý do GGUF_DEFAULT_ALIGNMENT = 32.

Q4_0: 32×418×8=128144=88.9%\frac{32 \times 4}{18 \times 8} = \frac{128}{144} = 88.9\%18×832×4​=144128​=88.9% (11.1% overhead cho scale).

4. Contiguity Conditions​

Tensor được gọi là contiguous (liên tục) khi:

nb[i]=nb[i−1]×ne[i−1]∀inb[i] = nb[i-1] \times ne[i-1] \quad \forall inb[i]=nb[i−1]×ne[i−1]∀i

Tensor transposed (hoán vị chiều 0 và 1) KHÔNG contiguous, nhưng GGML vẫn xử lý được nhờ stride-based access.

💡 Đúc kết​

Tensor layout trong GGML được thiết kế để tối ưu cho SIMD: 32-byte alignment đảm bảo vectorized loads đạt peak bandwidth, block packing tối thiểu overhead, và stride-based access cho phép xử lý transposed tensors mà không cần sao chép dữ liệu.
